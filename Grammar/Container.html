# 認識容器

# 容器是比較易懂的名稱，專業術語可以稱資料結構

容器就像是碗盤，資料型態的數據像是米飯、青菜等等

容器裡面的東西我們會稱其為元素

  1. 例如: 碗 = [ ]，菜和米飯 = 1、2、3、4
  2. EX: [1, 2, 3, 4, 5, 4]
  3. 上述範例就像是碗裡面裝了米飯 1 和菜 (2, 3, 4, 5)

了解各語言的內建容器

基本上主要會用到的是Dynamic Array和HashTable其餘用到機率比較低，因為Queue和Stack是可以透過Dynamic Array實現的

容器名稱 中文名稱 C C++ JavaScript Python
Array 陣列 { } { } 無 無
Dynamic Array 動態陣列 無 vector Array List
HashTable 雜湊表 無 map Map Dict
Set 集合(是數學上的集合) 無 set Set Set
Queue 佇列 無 queue 無(Array有功能可以達到其效果) 無(可以從內建函式庫匯入)
Stack 堆疊 無 Stack 無(Array有功能可以達到其效果) 無(可以從內建函式庫匯入)

認識下列容器

基本上以原理觀念為主而不是操作方式

# Array

# 陣列

特色

  1. 固定長度
  2. 有順序性
  3. 可以修改內容
  4. 代表連續性記憶體
  5. 透過index表示第幾個元素
  6. index 0 表示第1個元素
  7. 至於是0的原因如下
  8. 假設第一個位址是200 ，且資料型態是Int即4Bytes
  9. 則第一個元素位址是200
  10. 則第二個元素位址是204
  11. 公式如何第n個元素位址 = 200 + (n - 1) * 資料型態Size
  12. 所以第一個元素是0則位址公式變成 200 + n * 資料型態Size
  13. 也因此隨機存取速度非常快，只要用公式就可以取得位址不用慢慢尋找
  14. 所謂隨機存取是指隨便指定要讀取任意的元素所需要的速度
  15. 很多新語言都捨去Array只留Dynamic Array(所謂的簡化學習難度)

比如下圖

範例C++

int i[] = {1, 2, 3, 4, 5, 4, 6};
cout << i[0] << "\n"; // 印出第一個元素是 1
i[0] = 10; // 賦值第一個元素為10 (因為本來是1所以覆蓋過去)
cout << i[0] << "\n"; // 印出第一個元素是 10
cout << i[7] << "\n"; // 發生錯誤 超出Index長度

# Dynamic Array

# 動態陣列

特色

  1. 半固定長度
  2. 有順序性
  3. 可以修改內容
  4. 代表連續性記憶體
  5. 透過index表示第幾個元素 (原理同Array)
  6. 半固定長度是指
  7. 假設長度為8且裡面存放元素只有4個，則若要新增元素進去總長度不變，總長度8，元素5個，長度5
  8. 假設長度為8且裡面存放元素也是8個，則若要新增元素進去總長度會增加一倍，總長度16，元素9個，長度9

比如下圖

i = list()

i.append(20)
i.append(30)

print(i) # [10, 20, 30, 10, 40]
print(i[0]) # 10
i[0] = 70
print(i) # [70, 20, 30, 10, 40]
print(i[0]) # 70
i.append(60)
print(i) # [70, 20, 30, 10, 40, 60]

# HashTable

# 雜湊表

特色

  1. 透過Key尋找Value
  2. 因為是透過Key(關鍵字)取得Value(值)就好比人在查字典的時候，會帶入字根、部首、筆畫，找出其對應的文字，單字等等
  3. 無順序性 (順序性是透過Key尋找Value)
  4. 隨機存取很快
  5. 每個語言實作方式不太一樣
  6. 有很多別名比如Map(取自Mapping映射)，Dict(取自Dictionary字典)
  7. 隨機存取方式 Key帶入到一個HashFunction產生一個值，代表Value所在的位置，注意是位置不是位址，因為不是所有語言都是產生記憶體位址
  8. 因為產生的值會映射位置所以有些語言稱為Map
  9. 隨機存取的速度取決於HashFunction的複雜度
  10. HashFunction是A值帶進去必然產生B值，B值帶進去必然產生C值，即帶入的值不同產生不同的值，帶入的值一樣會產生相同的值
  11. 根據這個特性Key產生的值正常情況下不會和其他Key產生的值重疊(機率很低低到爆炸)
  12. 有些語言產生的值直接就是記憶體位址，而PyThon的值是產生Index
  13. PyThon的過程是 Key => [ HashFunciton ] => Index => [ List ] => Value
  14. 上述過程也只是簡略版，並非完整版本，但現階段認知程度到這邊就可以了
  15. 在程式競賽中Dict通常用於表示圖或是樹的連線關係，比如1號節點(Node)連向3號和4號Node而4號Node連向2號Node

圖例

Python範例 - 1

MyHashTable = dict()
MyHashTable["Key1"] = 50
MyHashTable["Key2"] = 10
MyHashTable["隔壁老王"] = "小三"

print(MyHashTable) # {'Key1': 50, 'Key2': 10, '隔壁老王': '小三'}

"""
我們可以發現是先印出越早定義的Key 而一組的結構是 Key : Value
"""

for i in MyHashTable:
    print(i, end = " ") # Key1 Key2 隔壁老王 
print()

"""
跑For的時候印出越早定義的Key
"""

for Key in MyHashTable:
    print(MyHashTable[Key], end = " ") # 50 10 小三
print()

"""
MyHashTable[key] 透過For讀取所有Key在帶入MyHashTable取得該Value
"""

MyHashTable[-10] = "欠錢"

"""
PyThon世界中Key和Value沒有限制Type，但這是屬於不太好的設計方式，弊大於利

型別管控是很重要的一件事情比如不知道該帶入10還是"10"

這種設計的好處基本上只剩下具有 optional arguments其中一種 keyword arguments 的function 在教函式的會講解

這種設計在實務上比較常見，因為是設計給團隊使用的函式，程式競賽基本上是不會用到的
"""

print(MyHashTable["Who"]) # KeyError 不能存取尚未定義Value的Key

# Set

# 集合(數學上定義的集合)

  1. 不會存在相同的元素
  2. 有些語言叫做HashSet
  3. 無順序性，更難聽一點無法指定讀取元素
  4. 允許迭代，這也是大多語言唯一確定裡面元素的方法
  5. 不懂就問數學老師，子母群體，可以求交集、聯集、差集等等關係(在高職數學的統計單元)
  6. 比如展開可能性，避免展開的該可能性之前展開過就會使用Set
  7. 比如求班上成績的多樣性，也會用Set去掉重複的成績

python 範例 (只有PyThon範例一次要打三個語言真的很累)

# Queue

# 佇列

特性

  1. 先進先出(後進後出)
  2. 有順序性，但不允許隨機訪問
  3. 存在所謂的讀取頭和寫出頭
  4. 每次讀取都是最後面的元素且會刪掉該資料專業術語彈出Pop或Dequeue
  5. 每次存入都是最前面的元素添加進去專業術語叫做PushBack或Enqueue

圖例

# Stack

# 堆疊

特性

  1. 先進後出(後進先出)
  2. 有順序性，但不允許隨機訪問
  3. 存在所謂的讀取頭和寫出頭
  4. 每次讀取都是最前面面的元素且會刪掉該資料專業術語彈出Pop
  5. 每次存入都是最前面的元素添加進去專業術語叫做Push

圖例

Queue和Stack就沒特別實作可以當作有讀取和寫入限制的Dynamic Array
