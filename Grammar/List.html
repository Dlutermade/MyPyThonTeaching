# List

回顧前章

我們都還記得PyThon的Dynamic Array是List，但所有語言List都是Dynamic Array嗎?

答案:不是

絕大部分高階程式語言都是使用Dynamic Array去實作List，但有極少部分使用Linked List(鏈結串列)去實作，Linked List優點就是Array(不是Dynamic Array)的缺點，其缺點就是Array的優點。

比如訪問第三個元素，Array只要帶Index公式就可以得到記憶體位址，但Linked List則必須從第一個元素慢慢往下找，因為Linked List位置是散開來的，第一個節點會記錄當前元素和下一個元素，這種設計優點是插入非常快，因為記憶體位址不需要一樣，而Array插入則必須先把該位子和其後面的原本資料往後移動(保持元素在記憶體是連續的)才可以插入比較浪費時間。

詳細可以Google ( Linked List )

建議盡量看圖解方式，程式碼實作未必會比較了解原理

1. 觀念 純粹動態語言的任何一切都是物件
這包含(JavaScript，Python)
函數(建構子，普通函數，Lambda)，類別等等全部都是物件，大陸稱呼物件為實例或對象
透過了解一切都是物件，推理出變數存放的值都是物件，那些名稱(list, int, float...)都是變數名

好處
這種設計可以把函數直接當作變數進行傳遞達到委託(函數篇時在提到)，變數可以存放一切東西

壞處
會直接覆蓋掉原先定義的函式，在其他語言變數名稱和函式名稱可以分開來不會產生覆蓋問題。

備註
非純動態語言的動態語言VB就不是了，C / C++ / C# 都是靜態語言自然不是。

結論
變數list的值從型別的建構子修改成int，也就是第五行做的事情

# 串列引用

# 引用聽起來很抽象，好聽一點說法，我們變數的值是不是同一個串列(引用同一個記憶體位址)

AList = []
BList = AList
print(id(AList), id(BList), id(AList) == id(BList)) 
# AList的值所在記憶體位址，BList的值記憶體所在位址，AList和BList是否同一個記憶體位址
AList.append("我是AList的添加物")
print(AList) # ['我是AList的添加物']
print(BList) # ['我是AList的添加物']
BList.append("我是BList的添加物")
print(AList) # ['我是AList的添加物', '我是BList的添加物']
print(BList) # ['我是AList的添加物', '我是BList的添加物']
""" 
第一段 AList = 創建一個List Object
第二段 BList = 引用AList 的 Object
所以AList和Blist是引用同個Object
所以凡是對這個Object的操作比如append都會影響AList和BList的值

專業說法:引用同個reference(參考) 即 引用同個記憶體位址，所以操作AList和BList都是操作同個記憶體位址

id(值) 回傳該值記憶體位址
