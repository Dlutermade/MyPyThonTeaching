# 程式的應用

這邊主要是淺談程式的應用和提供的學習資源，可以先大略看看，學完程式再回頭觀望一下

# 程式主要分兩大派系

  1. 看的到

通常是指Html和Css樣式上的設計

  1. 看不到

隱含在程式碼裡面的邏輯，比如按下A按鈕B會顯示、搜尋從A點走到B點的最佳路徑且路徑上不會有障礙物

# 前言

很多都是JavaScript語言的資源是因為有句諺語:任何東西都可以用C/C++寫，JavaScript可以寫任何應用

JavaScript也是最普及的語言之一，是上手簡單，想要超深入卻是所有語言最難的

原因C/C++是屬於可以直接寫底層，JavaScript社群很大，任何應用都可以讓JavaScript參一腳，但可能是JavaScript翻成別的語言去實現的

想學好使用JavaScript任何應用必須去學Node.js，因為除了在網頁上執行的JavaScript都是透過Node.js環境執行的，還有一種新的叫做Deno

有趣事項Node.js和Deno開發人是同一個人

Deno於2020年5月13日正式發布，開發人說: 是為了彌補11年前Node.js開發上所犯的錯誤(Node.js於2009年5月28日發布)

  1. 後端工程師

Python陣營

Python後端有兩大陣營，Flask派系和Django派系

  1. Flask派系比較適合新手學習當基礎，適合小型專案，因為功能不多，但很夠用，就像芭蕉一樣雖然小但比香蕉甜美
  2. Django派系比較適合碰過其他後端框架的人學習，適合大型專案，因為功能太多，甚至有些根本用不到，就像香蕉雖然很大，但未必比較甜美

作者個人平常是寫前後端分離Vue當前端，C#寫Api當後端，所以對作者來說Flask會比較好用，作者幾乎都是用C#用ASP.Net Core Api框架的

重點

  1. Flask: 彈性很高，Flask的概念是提供路由和渲染，其他部份可以匯入自己喜歡的套件去開發，整個專案的資料夾結構也比較隨興，簡單開發首選

入門簡單，學深難

  1. Django: 彈性很低，Django的概念是從路由、渲染、連接資料庫等等都是完整的，整個專案的資料夾結構也比較固定，因此運用上會比較吃後端所需要的知識

入門難，學深簡單

# 原生應用軟體工程師(原生應用軟體是指需要安裝且直接透過作業系統執行的軟體，而非原生是指需要安裝但不是直接透過作業系統執行的軟體)，通常原生應用效能都比較好，但俗話說的好夠用就可以了

  1. 原生應用是指需要安裝透過作業系統執行的軟體

Swift:開發ios應用程式(這是包含手機板ios和電腦版ios)

WinForm:透過C#或是VB.Net或是C++去開發Windows原生應用

WPF:透過C#或是VB.Net去開發Windows原生應用

WinForm上手比較簡單，但設計起來很難用，所以微軟又開發出WPF框架彌補設計上的難度

  1. 混和式透過其他語言引擎去翻譯成目標作業系統支援的功能或是透過其他語言引擎和作業系統的API溝通，達到極度接近原生應用的效能

React Native:可以使用JavaScript框架之一的React開發出安卓或ios應用(讓寫網頁的人也可以寫手機)

Flutter:透過JavaScript延伸的語言Dart即可開發出安卓或是ios應用(現代手機開發主流)

Xamarin:可以透過C#開發出安卓或是ios以及windows應用

Tkinter:可以透過Python開發出電腦介面程式，官方套件，但是非主流，因為效能很差原理是透過Tcl/Tk

想學學簡單PyThon桌面應用程式開發首選，但若注意設計就不要了巴

  1. 非原生應用是指透過網頁瀏覽器渲染引擎或是其他方式並非作業系統直接執行的軟體例如:Visual Studio Code、Discord桌面版、Twitch桌面版等等

Eel:堪稱Python版的Electron，且彈性最好透過Html和Css設計還可以撰寫JavaScript，且可以再PyThon和JavaScript之間互相呼叫函式，本質上是一個小型瀏覽器

Electron:透過JavaScript開發出桌面應用程式本質上是一個小型瀏覽器

Electron-Vue:透過Vue框架開發出桌面應用程式本質上是一個小型瀏覽器
