# 前言

很多都是JavaScript語言的資源是因為有句諺語:任何東西都可以用C/C++寫，JavaScript可以寫任何應用

JavaScript也是最普及的語言之一，是上手簡單，想要超深入卻是所有語言最難的

原因C/C++是屬於可以直接寫底層，JavaScript社群很大，任何應用都可以讓JavaScript參一腳，但可能是JavaScript翻成別的語言去實現的

想學好使用JavaScript任何應用必須去學Node.js，因為除了在網頁上執行的JavaScript都是透過Node.js環境執行的，還有一種新的叫做Deno

有趣事項Node.js和Deno開發人是同一個人

Deno於2020年5月13日正式發布，開發人說: 是為了彌補11年前Node.js開發上所犯的錯誤(Node.js於2009年5月28日發布)

# 網頁領域

Python後端有兩大陣營，Flask派系和Django派系

  1. Flask派系比較適合新手學習當基礎，適合小型專案，因為功能不多，但很夠用，就像芭蕉一樣雖然小但比香蕉甜美

  2. Django派系比較適合碰過其他後端框架的人學習，適合大型專案，因為功能太多，甚至有些根本用不到，就像香蕉雖然很大，但未必比較甜美

學長個人平常是寫前後端分離Vue當前端，C#寫Api當後端，所以對作者來說Flask會比較好用，作者幾乎都是用C#用ASP.Net Core Api框架的

...API 用途在於把資料傳給後端，後端做處理 比如資料驗證，資料儲存，資料計算，再把資料或是訊息傳給前端

  1. Flask: 彈性很高，Flask的概念是提供 路由 和 渲染 ，其他部份可以匯入自己喜歡的套件去開發，整個專案的資料夾結構也比較隨興，簡單開發首選

入門簡單且快，學深難 ( 因為很容易程式架構亂掉 )

  1. Django: 彈性很低，Django的概念是從路由、渲染、連接資料庫等等都是完整的，整個專案的資料夾結構也比較固定，因此運用上會比較吃後端所需要的知識

入門難且慢，學深簡單 ( 因為很多東西本來就有且限制多 )

簡單認識兩者差異

詳細認識兩者差異

在網頁領域近年來

Golang是取代Python在網頁後端領域的地位的

( 網頁領域 主流還是PHP的laravel)

因為Golang 語法簡潔度 和 Python 很像 且是靜態型別 不像Python動態型別且效能比Python快了整整快20倍了 甚至比Java還快 稍微接近C/C++效能

(效能來說 和C/C++最接近的是Rust語言 甚至Rust在某些情況下還比C/C++快 )

其中 Golang 主流之一 Gin 框架 就和 Flask很像

(Gin 在網頁框架效能 是除了名的)

主要原因是SPA網頁很適合搭配Flask和Gin這種輕量級框架

網頁框架人氣排行發展2012 ~ 2020
