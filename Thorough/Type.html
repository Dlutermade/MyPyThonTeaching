# 深入了解型別

在絕多數語言型別會分很多種，而每種型別會吃不同記憶體空間，講究的就是這空間與值的範圍拿捏技巧。

我們以 C++的 Type 作為範例

# 字的 Type

型別 中文 範圍 空間 作用 範例
char 字元 -128 ~ 127(-2^7 ~ 2^7 - 1) 1Byte 存放一個文字的 Ascii 數字碼的型別 'a'
string 字串 看語言(即使有上限也沒啥感覺) 1Byte + 長度(字數) *Char 的 Byte 由多個字元串接 "abcdefg"

string 的公式原理

string 是屬於 Reference(參考) Type，即指向記憶體的一個連續空間，所以紀錄記憶體起點的值是 1Byte，而接下來才是字數* Char 的 Byte，String 的訪問(access)方法是變數[index]，index 0 的時候是 (記憶體起點)0x005 + 0 * 1，index 2 的時候 (記憶體起點)0x005 + 2 * 1。

ㄋ

Char 的範圍

有些編譯器 Char 的範圍是 0 ~ 255(0 ~ 2^8 -1)版本不同也是會影響的，有些語言甚至會是 2Byte 範圍增加整整一倍，因為 Unicode 的 UTF-8 版本是 2Byte，這樣程式就支援跨語言使用即顯示中文同時兼容 Ascii 順序比如大寫 A 是 65。

PyThon 的關係

PyThon 省略了 Char 整合到 String 裡面了，PyThon2.X 是用 Ascii 編碼不支援中文，PyThon3.X 是用 UTF-8 編碼支援跨語言。

# 整數的 Type

型別 中文 範圍 空間 作用
short 短整數 -32768 ~ 32767(-2^15 ~ 2^15 - 1) 2Byte 存放整數值
unsigned short 短正整數 0 ~ 65535(0 ~ 2 ^ 16 - 1) 2Byte 存放正整數值
int 整數 -2^31 ~ 2^31 - 1 4Byte 存放整數值
unsigned int 正整數 0 ~ 2 ^ 32 - 1 4Byte 存放正整數值
long long 長整數 -2^63 ~ 2^63 - 1 4Byte 存放整數值(一個 long 是 int 兩個 long 才是 long)
unsigned long long 長正整數 0 ~ 2 ^ 64 - 1 8Byte 存放正整數值

上述空間與範圍並非絕對

C++還是會受到版本影響，所以並非絕對，上述是屬於 64 位元系統且較大多數語言都是採用這套版本計算的，比如 32 位元系統 Long 和 Int 都是 4Byte 但是 Long 範圍始終都會大於 Int。

空間不夠用怎某辦

C++最大可以到 2 ^ 64 - 1 假設這樣都不夠用會採用手寫大整數(Big Int)Type，利用動態陣列(資料結構會講)達到類似於桶子概念，當一個桶子滿了(即數字塞不下)，為 Type 添加桶子，改塞下一個桶子。

很少程式競賽題目會針對 Big Int 去考 (即題目變化不高)

PyThon 的關係

PyThon 的整數就是 Big Int 即大整數 Type，會自動擴充，但大到一定的程度依然會塞不下，具體實作方式這要看 PyThon 版本和官方文檔，但我們只需要知道原理即可。

# 浮點數的 Type

型別 中文 範圍 空間 作用 範例
float 浮點數 3.4E +/- 38 (7 位數) 4Byte 存放單精度浮點數 3.14159
double 雙精度浮點數 1.7E +/- 308 (15 位數) 8Byte 存放雙精度浮點數 3.1415926535897

盡量避免使用浮點數來加減

在網頁專用語言 JavaScript 有個經典面試題目叫做 0.1 + 0.2 是否等於 0.3
答案是否的因為 0.1 + 0.2 = 0.30000000000000004，因為電腦計算是先轉二進位在轉回來十進位的

詳細請 Google (IEEE 754 Floating-point)

PyThon 的關係

PyThon 的 float 就是 C++的 double
